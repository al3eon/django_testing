# Тестирование Django проектов
Проекты демонстрируют подходы к тестированию веб-приложений на Django с использованием двух популярных фреймворков.

## YaNote - тестирование на unittest
### Настройка тестов:
- Тесты расположены в модулях test_routes.py, test_content.py, test_logic.py
- Используется встроенный фреймворк Django unittest
- Применяются TestCase и Client для эмуляции запросов
- Покрытие тестами:
- Маршруты и права доступа
- Корректность передаваемого контекста

### Запуск:
```bash
python manage.py test
```

## YaNews - тестирование на pytest
### Настройка тестов:
- Тесты организованы в тех же модулях
- Используется фреймворк pytest и pytest-django
- Фикстуры для создания тестовых данных

### Покрытие тестами:
- Доступность эндпоинтов
- Логика отображения контента
- Работа с формами и валидация

### Запуск:
```bash
pytest
```

## Структура тестов
Оба проекта содержат одинаковые группы тестов:

### Тесты маршрутов (routes)
- Проверка доступности страниц
- Тестирование редиректов
- Проверка прав доступа

### Тесты контента (content)
- Проверка передаваемого контекста
- Тестирование сортировки и фильтрации
- Проверка лимитов отображения

### Тесты логики (logic)
- Тестирование бизнес-логики
- Проверка валидации данных
- Тестирование прав доступа к операциям
